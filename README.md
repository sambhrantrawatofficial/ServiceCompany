# Contributing Guide

## Branching
- `main`: protected, release-ready
- `develop`: integration
- `feature/*`: scoped features
- `fix/*`: hotfixes

## Commit convention (Conventional Commits)
- `feat: add orders module`
- `fix: booking date validation`
- `docs: update consumer policy`
- `chore: bump dependencies`

## PR checklist
- Tests pass locally
- Follows code style
- Updated docs (README/policies)
- Screenshots for UI changes
- Linked issue

## Code style
- .NET: analyzers enabled, nullable reference types
- Razor: semantic HTML, Bootstrap grid
- SQL: migrations, no raw queries without parameters
